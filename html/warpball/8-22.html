Warpball Development Log

August 22nd 2021

Changes: 
- Added level indicator at top of screen (Game)
- Game Menu
    - Reworked Main Menu scene to have 3D component
        - Camera on Dolly track rotating around player model
    - Skybox added
    - Blurred UI components
    - History/Statistics
        - Basic working run history/statistics system (uses PlayerPrefs as persistent storage)
            - persistent storage scripting system/interface 
        - history and stats window
            - sort by different categories in history (time, score, level)
        - temporary transition animations (script animations)
- Game Over Menu
    - Basic Game Over menu implemented
        - Incrementing score display (towards final score)
    - New game/Return to menu button
    - Script animated instantiate animation
- Game appearance
    - Added game over player crash "animation" (activate physics/rigidbodies)
    - Adjusted player jet particle prefabs
    - UI separated to different camera
        - UI specific post processing added (Chromatic abberation, lens distortion)
    - Runtime post processing modification system added 
        - Boost post processing effects
    - Added space skybox (free from unity asset store) 
    - Adjusted collectible and ramp prefabs 
        - Point lights added to collectible prefabs 
        - Added light to ramp prefab
    - Added fog
- Gameplay 
    - Boost now costs 100% of boost charge
        - Allows players to break obstacles instead of crashing during boost time
        - activates only when (space bar) is pressed on 100% boost charge
    - Added hold boost - activates when pressed on < 100% boost charge
        - Gives player more speed, but adds points during hold boost as reward
    - Strafe boost now costs 0
- Performance
    - Environment generation zone size decreased dramatically
        - ↑ frequency of environment generation frames 
        - ↓ simultaneous gameobjects (~1500 -> ~400)
        - ↓ max frame time (~70ms -> ~30ms)
    - Environment generation now occurs over multiple frames (as opposed to over a single frame)
        - one frame per type of object
        - lag spike less apparent
    - disabled v-sync
- Other
    - Moved all boost script logic in player controller to boost controller
    - simplified boost logics - use LeanTween over custom SmoothDamp logic
    - added debug display in-game
        - fps counter, game object number counter
    - player velocity now preserves while inAir
        - player velocity gradually slows after landing (instead of instantly slowing)
- Bugs
    - fixed inconsistent main menu lighting - ref: 
      (https://forum.unity.com/threads/loading-scene-additively-causes-change-in-lighting.511566/)
    - fixed end game menu not triggerin consistently
    - fixed player moving even after crash
